  <name>Task 1: Create communication tables migration with Realtime</name>
  <files>supabase/migrations/20260305000003_communication.sql</files>
  <action>
Create `supabase/migrations/20260305000003_communication.sql` containing:

**1. chat_messages table:**
- `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
- `organization_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE`
- `sender_id UUID REFERENCES profiles(id)` — nullable (AIDEAS-side messages may not have a user id)
- `sender_type TEXT NOT NULL CHECK (sender_type IN ('client', 'aideas'))` — no FK for AIDEAS side
- `content TEXT NOT NULL` — text-only for v1 (no file attachments per CONTEXT.md)
- `created_at TIMESTAMPTZ DEFAULT NOW()`
- No `updated_at` or `deleted_at` (chat messages are immutable — per CONTEXT.md, no edit/delete in v1)
- Enable RLS
- SELECT policy: org-scoped via EXISTS on organization_members, `(SELECT auth.uid())`
- INSERT policy: `WITH CHECK` verifying `sender_type = 'client'` AND `sender_id = (SELECT auth.uid())` AND org membership via EXISTS on organization_members (per CONTEXT.md: interaction tables allow user INSERT; AIDEAS-side messages sent via service_role)
- No UPDATE/DELETE policies
- Indexes: `organization_id`, `created_at` (for chronological message retrieval)

**2. Enable Realtime on chat_messages:**
```sql
ALTER PUBLICATION supabase_realtime ADD TABLE chat_messages;
```
Per CONTEXT.md: Realtime enabled ONLY on chat_messages. Notifications and other changes via polling/page load.

**3. notifications table:**
- `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
- `organization_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE`
- `user_id UUID NOT NULL REFERENCES profiles(id)` — notification recipient
- `type TEXT NOT NULL CHECK (type IN ('info', 'warning', 'success', 'action_required'))` — per CONTEXT.md (CHECK constraint, not ENUM)
- `title VARCHAR(255) NOT NULL`
- `message TEXT`
- `is_read BOOLEAN NOT NULL DEFAULT false`
- `read_at TIMESTAMPTZ`
- `link TEXT` — optional URL to navigate to when notification is clicked
- `created_at TIMESTAMPTZ DEFAULT NOW()`
- No `deleted_at` (notifications are never soft-deleted; old ones are just read)
- Enable RLS
- SELECT policy: `user_id = (SELECT auth.uid())` — user sees only their own notifications
- UPDATE policy: `user_id = (SELECT auth.uid())` — user can mark as read (update is_read and read_at)
- No INSERT/DELETE policies (service_role creates notifications)
- Indexes: `organization_id`, `user_id`, composite `(user_id, is_read)` (for unread count queries)

**4. invitations table:**
- `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
- `organization_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE`
- `email VARCHAR(255) NOT NULL`
- `role TEXT NOT NULL DEFAULT 'viewer' CHECK (role IN ('admin', 'operator', 'viewer'))`
- `token TEXT UNIQUE NOT NULL DEFAULT encode(gen_random_bytes(32), 'hex')` — 64-char hex token per research
- `invited_by UUID NOT NULL REFERENCES profiles(id)`
- `expires_at TIMESTAMPTZ NOT NULL` — application sets, typically 7 days
- `accepted_at TIMESTAMPTZ`
- `created_at TIMESTAMPTZ DEFAULT NOW()`
- No `deleted_at` (invitations expire, not soft-delete)
- Enable RLS
- SELECT policy: org-scoped via EXISTS on organization_members, `(SELECT auth.uid())`
- No INSERT/UPDATE/DELETE policies (service_role manages invitations)
- Indexes: `token`, `organization_id`, `email`

**IMPORTANT PATTERNS:**
- All `auth.uid()` MUST be wrapped as `(SELECT auth.uid())`
- All policies specify `TO authenticated`
- CHECK constraints for all type/role columns
- chat_messages INSERT uses `WITH CHECK` (not `USING`) and validates both sender_id and org membership
