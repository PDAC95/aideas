<objective>
Build the runtime translations infrastructure that makes Phase 18 admin edits visible on the customer side without a redeploy. Adds the `automation_template_translations` table, backfills it from existing `messages/{en,es}.json#templates`, and refactors the customer catalog (`/dashboard/catalog` and `/dashboard/catalog/[slug]`) to read display strings from the new table instead of bundled JSON.

Purpose: Next.js bundles `messages/*.json` at build time, so without this refactor any future admin edit (Plans 18-02/18-03) would require a redeploy to surface. This plan delivers the data plane; subsequent plans deliver the admin UI on top.

Output:
- New migration `20260507000001_template_translations.sql` with table + RLS + admin policies + backfill
- Updated `CatalogTemplate` / `CatalogTemplateDetail` shapes carrying resolved display strings
- Refactored `fetchCatalogTemplates` / `fetchTemplateBySlug` queries with locale-aware JOIN
- Customer catalog pages stripped of `getTranslations("templates")` for template-specific copy (chrome translations like "Mas populares" still come from `dashboard.catalog`)
</objective>
