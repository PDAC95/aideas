<objective>
Build the write side of catalog admin: dedicated `/admin/catalog/new` and `/admin/catalog/[slug]/edit` pages backed by a single-page grouped form. Operators can create new templates from scratch (entering all 13 fields plus 8 bilingual texts) or edit any field on an existing template. Slug is auto-generated from the EN name and locked after first save. Industries render as togglable chips, connected_apps as a multi-select dropdown. Strict Zod validation enforces all required fields when `is_active=true`; setting `is_active=false` (draft mode) bypasses required-field enforcement so partial work can be saved.

Purpose: closes the loop on CTLG-02 (create) and CTLG-03 (edit). Together with Plan 18-02, ops can fully manage the catalog through the admin UI without touching `seed.sql`.

Output:
- Two new admin pages: `/admin/catalog/new` and `/admin/catalog/[slug]/edit`
- One shared `<AdminTemplateForm />` client component
- Two server actions (`createTemplate`, `updateTemplate`) handling templates + translations atomically
- Zod validation schema with conditional refinement on `is_active`
- New `admin.catalog.form.*` i18n namespace at full EN/ES parity
</objective>

Step A. Create `web/src/lib/validations/admin-catalog-template.ts`.

Export Zod schemas:

```typescript
const SLUG_RE = /^[a-z0-9]+(-[a-z0-9]+)*$/;
const CATEGORIES = ['customer_service','documents','marketing','sales','operations','productivity','reports','ai_agents'] as const;
const INDUSTRIES = ['retail','salud','legal','inmobiliaria','restaurantes','agencias'] as const;
const PRICING_TIERS = ['starter','pro','business'] as const;

// Base — applies in both draft and active mode. Slug + meta fields always required.
export const adminCatalogTemplateBaseSchema = z.object({
  slug: z.string().min(2).max(100).regex(SLUG_RE, 'invalid_slug'),
  is_active: z.boolean(),
  is_featured: z.boolean(),
  sort_order: z.coerce.number().int().min(0),

  // Translations — strings, can be empty in draft mode
  name_en: z.string().max(255),
  name_es: z.string().max(255),
  description_en: z.string(),
  description_es: z.string(),
  typical_impact_text_en: z.string(),
  typical_impact_text_es: z.string(),
  activity_metric_label_en: z.string().max(100),
  activity_metric_label_es: z.string().max(100),

  // Categorization + lists
  category: z.enum(CATEGORIES),
  industry_tags: z.array(z.enum(INDUSTRIES)).default([]),
  connected_apps: z.array(z.string().min(1)).default([]),

  // Pricing (cents)
  pricing_tier: z.enum(PRICING_TIERS),
  setup_price: z.coerce.number().int().min(0).nullable(),
  monthly_price: z.coerce.number().int().min(0).nullable(),

  // Metrics
  setup_time_days: z.coerce.number().int().min(1).max(30).nullable(),
  avg_minutes_per_task: z.coerce.number().int().min(1).max(480).nullable(),

  // Optional
  icon: z.string().max(50).optional().nullable(),
});

// Active — refines base to require all fields that are nullable/empty-allowed in draft
export const adminCatalogTemplateActiveSchema = adminCatalogTemplateBaseSchema.superRefine((data, ctx) => {
  if (!data.is_active) return; // draft mode: skip activation-required checks
  const required = [
    ['name_en', 1], ['name_es', 1],
    ['description_en', 1], ['description_es', 1],
    ['typical_impact_text_en', 1], ['typical_impact_text_es', 1],
    ['activity_metric_label_en', 1], ['activity_metric_label_es', 1],
  ] as const;
  for (const [field, min] of required) {
    if ((data as any)[field].length < min) {
      ctx.addIssue({ path: [field], code: 'custom', message: 'required_when_active' });
    }
  }
  for (const field of ['setup_price','monthly_price','setup_time_days','avg_minutes_per_task'] as const) {
    if (data[field] == null) {
      ctx.addIssue({ path: [field], code: 'custom', message: 'required_when_active' });
    }
  }
  if (data.industry_tags.length === 0) {
    ctx.addIssue({ path: ['industry_tags'], code: 'custom', message: 'pick_at_least_one' });
  }
});

export type AdminCatalogTemplateInput = z.infer<typeof adminCatalogTemplateBaseSchema>;
```

Notes:
- `setup_price` / `monthly_price` accept null in draft mode (Zod `nullable()`); active mode requires non-null via the superRefine.
- The form sends prices in CENTS (the form layer multiplies dollars * 100 before submit). The schema sees integers.

```typescript
export type CreateTemplateResult =
  | { ok: true; slug: string }
  | { ok: false; error: 'invalid_input' | 'slug_taken' | 'not_authenticated' | 'not_staff' | 'create_failed'; fieldErrors?: Record<string, string> };

export async function createTemplate(input: AdminCatalogTemplateInput): Promise<CreateTemplateResult>;

export type UpdateTemplateResult =
  | { ok: true }
  | { ok: false; error: 'invalid_input' | 'not_found' | 'not_authenticated' | 'not_staff' | 'update_failed'; fieldErrors?: Record<string, string> };

export async function updateTemplate(slug: string, input: AdminCatalogTemplateInput): Promise<UpdateTemplateResult>;
```

`createTemplate`:
1. `assertPlatformStaff` first; on failure return error.
2. Choose schema: `is_active ? adminCatalogTemplateActiveSchema : adminCatalogTemplateBaseSchema`. `safeParse(input)`. On failure, flatten zod issues to `{ [field]: 'required_when_active' | 'invalid_slug' | ... }` and return `{ ok: false, error: 'invalid_input', fieldErrors }`.
3. Pre-flight uniqueness: `select id from automation_templates where slug = $1`. If a row exists, return `{ ok: false, error: 'slug_taken' }`.
4. Build the `automation_templates` row:
   - `name`: legacy i18n key shape `templates.${slug.replaceAll('-','_')}.name` (backward compat per CONTEXT.md spec)
   - `description`: copy from `description_en` (fallback snapshot)
   - `typical_impact_text`: copy from `typical_impact_text_en`
   - `activity_metric_label`: copy from `activity_metric_label_en`
   - All other 12 columns from input
5. INSERT and select id back.
6. INSERT 8 rows into `automation_template_translations` (one per locale x field). Build a single multi-row insert.
7. If translation insert fails: DELETE the just-created template row to avoid orphans (`.from('automation_templates').delete().eq('id', newId)`); return `{ ok: false, error: 'create_failed' }`.
8. `revalidatePath('/admin/catalog'); revalidatePath('/dashboard/catalog');`
9. Return `{ ok: true, slug }`.

`updateTemplate`:
1. `assertPlatformStaff` first.
2. Same schema selection + parse.
3. Look up template by slug: `select id from automation_templates where slug = $1`. If not found, `{ ok: false, error: 'not_found' }`.
4. UPDATE `automation_templates` with all 12 mutable fields (do NOT change slug — it is locked per CONTEXT.md). Mirror the same legacy-key snapshot logic for the 4 legacy text columns (sync to EN values).
5. UPSERT 8 translation rows: `.from('automation_template_translations').upsert(rows, { onConflict: 'template_id,locale,field' })`.
6. On any failure return `{ ok: false, error: 'update_failed' }`.
7. `revalidatePath('/admin/catalog'); revalidatePath('/dashboard/catalog'); revalidatePath(\`/admin/catalog/\${slug}/edit\`);`
8. Return `{ ok: true }`.
