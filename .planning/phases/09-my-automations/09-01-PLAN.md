**Queries (add to queries.ts):**

1. `fetchAutomationsPage(orgId: string)` — returns `AutomationsPageAutomation[]`:
   - Query automations with template join: `id, name, status, last_run_at, template:automation_templates(category, connected_apps, activity_metric_label, avg_minutes_per_task, monthly_price)`
   - Filter: `.eq("organization_id", orgId).is("deleted_at", null).not("status", "eq", "archived")`
   - In parallel, query automation_executions for monthly counts: `.select("automation_id").eq("status", "success").gte("started_at", monthStart)` scoped to org automation IDs via `.in("automation_id", orgAutomationIds)`
   - Use `Promise.all` for parallel fetching (automations first to get IDs, then monthly execs)
   - Compute `monthly_execution_count` per automation from execution data
   - Sort: active first, in_setup second, paused third (use STATUS_ORDER map), alphabetical within groups
   - Return sorted array

2. `fetchAutomationDetail(automationId: string, orgId: string)` — returns `{ automation, executions, weeklyData, monthlyMetricCount, hoursSaved }`:
   - Query single automation: `.eq("id", automationId).eq("organization_id", orgId).single()` with template join
   - Query last 20 executions: `.eq("automation_id", automationId).order("started_at", { ascending: false }).limit(20)` — select id, status, started_at, completed_at, duration_ms, error_message
   - Query 28-day executions for weekly chart: `.eq("automation_id", automationId).eq("status", "success").gte("started_at", fourWeeksAgo)` — select started_at, status
   - Query monthly metric count: `.eq("automation_id", automationId).eq("status", "success").gte("started_at", monthStart)` — select with count exact head true
   - Use `Promise.all` for all 4 queries in parallel
   - Compute `weeklyData: WeeklyChartData[]` — 4 buckets, group by week, most recent = last bucket. Use `groupByWeek` helper.
   - Compute `hoursSaved` from monthly count * avg_minutes_per_task / 60 (rounded to 1 decimal)
   - Return `{ automation, executions, weeklyData, monthlyMetricCount, hoursSaved }`

**Helper function (in queries.ts, not exported or exported):**

`groupByWeek(executions: { started_at: string }[], now: Date): WeeklyChartData[]` — groups executions into 4 weekly buckets, returns array with labels "W1" through "W4" (W4 = most recent week).

**Important notes:**
- Follow established pattern: `createClient()` from `@/lib/supabase/server` at start of each function
- Use `.in("automation_id", orgAutomationIds)` pattern for scoping (established in 08-01, avoid nested relation filters)
- `monthStart` = first day of current month, `fourWeeksAgo` = 28 days before now
- Cast Supabase results with `as unknown as Type[]` pattern (established in queries.ts)
