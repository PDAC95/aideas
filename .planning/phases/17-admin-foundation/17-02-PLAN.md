<objective>
Build the auth boundary that lets staff sign in independently from customers and lets admin server actions verify the caller. This plan establishes:

1. **Two-cookie session scheme** — customer sessions use Supabase's default `sb-*` cookies; staff sessions use a separate `sb-admin-*` cookie set so both can coexist in one browser (Stripe / Supabase / Vercel UX).
2. **Middleware gate** — `/admin/*` requires staff; `/dashboard/*` redirects staff to `/admin`; `/admin/login` is the unauthenticated landing for staff.
3. **`assertPlatformStaff(role?)` helper** — typed return for server actions, mirrors `assertOrgMembership`.

Purpose: Without this plan, `/admin/*` is either inaccessible or trivially bypassable, and admin server actions in Phases 18-22 have no shared way to assert "the caller is staff."

Output: Two new Supabase client factories, an extended middleware, the helper, the `/admin/login` page + form, and the sign-in/out server actions for staff.
</objective>

**File 1 — `web/src/lib/supabase/admin-server.ts`:**

Mirror `web/src/lib/supabase/server.ts` but configure `createServerClient` with a custom `cookieOptions.name` so admin cookies live under a different namespace. Supabase's `@supabase/ssr` accepts a `cookieOptions: { name: 'sb-admin' }` (the default is `sb-<project-ref>`). The cookie API still does getAll/setAll, but only cookies whose name starts with the configured name prefix are read/written.

Implementation:

```typescript
import { createServerClient } from "@supabase/ssr";
import { cookies } from "next/headers";

const ADMIN_COOKIE_NAME = "sb-admin";

export async function createAdminServerClient() {
  const cookieStore = await cookies();

  return createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookieOptions: { name: ADMIN_COOKIE_NAME },
      cookies: {
        getAll() {
          return cookieStore.getAll().filter((c) => c.name.startsWith(ADMIN_COOKIE_NAME));
        },
        setAll(cookiesToSet) {
          try {
            cookiesToSet.forEach(({ name, value, options }) =>
              cookieStore.set(name, value, options)
            );
          } catch {
            // Called from a Server Component — ignored when middleware refreshes session.
          }
        },
      },
    }
  );
}

export const ADMIN_SUPABASE_COOKIE_PREFIX = ADMIN_COOKIE_NAME;
```

**File 3 — `web/src/lib/auth/assert-platform-staff.ts`:**

Mirror the style of `assert-org-membership.ts` but return a discriminated union with explicit ok branch:

```typescript
import type { SupabaseClient } from "@supabase/supabase-js";

export type StaffRole = "super_admin" | "operator";

export type AssertPlatformStaffResult =
  | { ok: true; userId: string; role: StaffRole }
  | { ok: false; error: "not_authenticated" }
  | { ok: false; error: "not_staff" }
  | { ok: false; error: "insufficient_role" };

/**
 * Verifies the authenticated caller is in platform_staff.
 *
 * @param supabase - admin-scoped Supabase client (createAdminServerClient())
 * @param requiredRole - if provided, caller must have exactly this role
 *
 * Returns ok=true with userId+role on success, or ok=false with a typed error.
 * Logs structured console.error on rejection.
 */
export async function assertPlatformStaff(
  supabase: SupabaseClient,
  requiredRole?: StaffRole
): Promise<AssertPlatformStaffResult> {
  const {
    data: { user },
    error: authError,
  } = await supabase.auth.getUser();

  if (authError || !user) {
    console.error("[assertPlatformStaff] not_authenticated");
    return { ok: false, error: "not_authenticated" };
  }

  const { data: staff } = await supabase
    .from("platform_staff")
    .select("role")
    .eq("user_id", user.id)
    .single();

  if (!staff) {
    console.error("[assertPlatformStaff] not_staff", { userId: user.id });
    return { ok: false, error: "not_staff" };
  }

  const role = staff.role as StaffRole;
  if (requiredRole && role !== requiredRole) {
    console.error("[assertPlatformStaff] insufficient_role", {
      userId: user.id,
      have: role,
      need: requiredRole,
    });
    return { ok: false, error: "insufficient_role" };
  }

  return { ok: true, userId: user.id, role };
}
```

**`web/src/lib/supabase/middleware.ts`:**

Refactor `updateSession` so that:

1. The path-based branch happens FIRST. If `pathname.startsWith('/admin')` (and is NOT `/admin/login`), use the admin cookie scope; otherwise use the existing customer scope.
2. Both cookie scopes can be present simultaneously — the Supabase client only reads its own prefix, so they don't fight.
3. Add `/admin/*` rules:
   - Unauthenticated + path starts with `/admin` (and not `/admin/login`) → redirect to `/admin/login`.
   - Authenticated as admin user but NOT a platform_staff member → redirect to `/admin/login` with `?error=not_staff` (do NOT redirect to `/dashboard` — that would mix sessions).
   - Authenticated admin already on `/admin/login` → redirect to `/admin`.
4. Add the cross-redirect: when a request hits `/dashboard` and the customer-scope user IS a `platform_staff` member, redirect to `/admin`. This handles the case where a staff member accidentally lands on the customer area.

Implementation outline (full file):

```typescript
import { createServerClient } from "@supabase/ssr";
import { NextResponse, type NextRequest } from "next/server";

const ADMIN_COOKIE_PREFIX = "sb-admin";

export async function updateSession(request: NextRequest) {
  const pathname = request.nextUrl.pathname;
  const isAdminPath = pathname.startsWith("/admin");
  const isAdminLogin = pathname === "/admin/login";

  let supabaseResponse = NextResponse.next({ request });

  // Customer remember-me preference (unchanged)
  const rememberMe = request.cookies.get("sb-remember-me")?.value;
  const customerCookieMaxAge = rememberMe === "false" ? undefined : 60 * 60 * 24 * 30;

  // ---- Build the customer client (default cookie scope) ----
  const customerSupabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return request.cookies.getAll().filter((c) => !c.name.startsWith(ADMIN_COOKIE_PREFIX));
        },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value }) => request.cookies.set(name, value));
          supabaseResponse = NextResponse.next({ request });
          cookiesToSet.forEach(({ name, value, options }) =>
            supabaseResponse.cookies.set(name, value, { ...options, maxAge: customerCookieMaxAge })
          );
        },
      },
    }
  );

  // ---- Build the admin client (sb-admin-* cookie scope) ----
  const adminSupabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookieOptions: { name: ADMIN_COOKIE_PREFIX },
      cookies: {
        getAll() {
          return request.cookies.getAll().filter((c) => c.name.startsWith(ADMIN_COOKIE_PREFIX));
        },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value }) => request.cookies.set(name, value));
          supabaseResponse = NextResponse.next({ request });
          cookiesToSet.forEach(({ name, value, options }) =>
            supabaseResponse.cookies.set(name, value, { ...options })
          );
        },
      },
    }
  );

  // Resolve users in both scopes (parallel — no DB join needed)
  const [{ data: { user: customerUser } }, { data: { user: adminUser } }] = await Promise.all([
    customerSupabase.auth.getUser(),
    adminSupabase.auth.getUser(),
  ]);

  // ---- /admin/* gate ----
  if (isAdminPath) {
    if (!adminUser) {
      if (!isAdminLogin) {
        const url = request.nextUrl.clone();
        url.pathname = "/admin/login";
        return NextResponse.redirect(url);
      }
      return supabaseResponse;
    }

    // adminUser exists — verify they're staff
    const { data: staff } = await adminSupabase
      .from("platform_staff")
      .select("user_id")
      .eq("user_id", adminUser.id)
      .single();

    if (!staff) {
      // Logged in as a non-staff user via admin scope — sign them out and bounce
      await adminSupabase.auth.signOut();
      const url = request.nextUrl.clone();
      url.pathname = "/admin/login";
      url.searchParams.set("error", "not_staff");
      return NextResponse.redirect(url);
    }

    if (isAdminLogin) {
      // Already authenticated staff visiting login → go to /admin
      const url = request.nextUrl.clone();
      url.pathname = "/admin";
      return NextResponse.redirect(url);
    }

    return supabaseResponse;
  }

  // ---- Existing customer-area logic (preserved) ----
  if (!customerUser && (pathname.startsWith("/dashboard") || pathname.startsWith("/app/"))) {
    const url = request.nextUrl.clone();
    url.pathname = "/login";
    return NextResponse.redirect(url);
  }

  if (
    customerUser &&
    !customerUser.email_confirmed_at &&
    !pathname.startsWith("/verify-email") &&
    !pathname.startsWith("/auth/") &&
    (pathname.startsWith("/dashboard") || pathname.startsWith("/app/"))
  ) {
    const url = request.nextUrl.clone();
    url.pathname = "/verify-email";
    url.searchParams.set("email", customerUser.email ?? "");
    return NextResponse.redirect(url);
  }

  // ---- NEW: staff on /dashboard → redirect to /admin ----
  if (customerUser && pathname.startsWith("/dashboard")) {
    const { data: staff } = await customerSupabase
      .from("platform_staff")
      .select("user_id")
      .eq("user_id", customerUser.id)
      .single();
    if (staff) {
      const url = request.nextUrl.clone();
      url.pathname = "/admin";
      return NextResponse.redirect(url);
    }
  }

  if (customerUser && (pathname === "/login" || pathname === "/signup")) {
    const url = request.nextUrl.clone();
    url.pathname = "/dashboard";
    return NextResponse.redirect(url);
  }

  if (pathname === "/") {
    if (customerUser) {
      const url = request.nextUrl.clone();
      url.pathname = "/dashboard";
      return NextResponse.redirect(url);
    }
    return NextResponse.rewrite(new URL("/landing/index.html", request.url));
  }

  return supabaseResponse;
}
```

**File 3 — `web/src/lib/actions/admin-auth.ts`:**

Two server actions: `signInStaff` and `signOutStaff`. Both write to the admin-scoped cookie set (use `createAdminServerClient`). On successful sign-in, double-check the user is in `platform_staff`; if not, sign them out and return `{ error: "not_staff" }` so the form shows the friendly message.

```typescript
"use server";

import { createAdminServerClient } from "@/lib/supabase/admin-server";
import { redirect } from "next/navigation";

export async function signInStaff(formData: FormData): Promise<{ error: string } | undefined> {
  const email = String(formData.get("email") ?? "").trim();
  const password = String(formData.get("password") ?? "");

  if (!email || !password) {
    return { error: "Email and password are required." };
  }

  const supabase = await createAdminServerClient();

  const { data, error } = await supabase.auth.signInWithPassword({ email, password });
  if (error || !data.user) {
    return { error: "Invalid credentials." };
  }

  const { data: staff } = await supabase
    .from("platform_staff")
    .select("user_id")
    .eq("user_id", data.user.id)
    .single();

  if (!staff) {
    await supabase.auth.signOut();
    return { error: "This account is not registered as platform staff." };
  }

  redirect("/admin");
}

export async function signOutStaff() {
  const supabase = await createAdminServerClient();
  await supabase.auth.signOut();
  redirect("/admin/login");
}
```
