**Section 1 — platform_staff table:**

```sql
CREATE TABLE IF NOT EXISTS public.platform_staff (
    user_id     UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
    role        TEXT NOT NULL CHECK (role IN ('super_admin', 'operator')),
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

ALTER TABLE public.platform_staff ENABLE ROW LEVEL SECURITY;

-- SELECT for staff themselves so client code can detect "am I staff?" without service role.
-- Writes (INSERT/UPDATE/DELETE) are deliberately service_role only — staff invitation UI is deferred (see CONTEXT.md "Deferred Ideas").
DROP POLICY IF EXISTS "platform_staff_select_self" ON public.platform_staff;
CREATE POLICY "platform_staff_select_self"
    ON public.platform_staff
    FOR SELECT
    TO authenticated
    USING (user_id = (SELECT auth.uid()));

CREATE INDEX IF NOT EXISTS idx_platform_staff_role ON public.platform_staff (role);
```

**Section 2 — Helper functions (SECURITY DEFINER):**

```sql
-- SECURITY DEFINER so callers without RLS access to platform_staff can still
-- check membership. SET search_path = '' to prevent schema-hijack attacks.
CREATE OR REPLACE FUNCTION public.is_platform_staff(uid UUID)
RETURNS BOOLEAN
LANGUAGE sql
SECURITY DEFINER
SET search_path = ''
STABLE
AS $$
    SELECT EXISTS (
        SELECT 1 FROM public.platform_staff WHERE user_id = uid
    );
$$;

CREATE OR REPLACE FUNCTION public.is_super_admin(uid UUID)
RETURNS BOOLEAN
LANGUAGE sql
SECURITY DEFINER
SET search_path = ''
STABLE
AS $$
    SELECT EXISTS (
        SELECT 1 FROM public.platform_staff
        WHERE user_id = uid AND role = 'super_admin'
    );
$$;

GRANT EXECUTE ON FUNCTION public.is_platform_staff(UUID) TO authenticated;
GRANT EXECUTE ON FUNCTION public.is_super_admin(UUID) TO authenticated;
```

**Section 3 — Seed first super_admin (idempotent + safe-no-op):**

```sql
DO $$
DECLARE
    v_user_id UUID;
BEGIN
    SELECT id INTO v_user_id FROM auth.users WHERE email = 'pdmckinster@gmail.com' LIMIT 1;
    IF v_user_id IS NULL THEN
        RAISE NOTICE 'platform_staff seed: pdmckinster@gmail.com not found in auth.users — skipping. Re-run this migration after the user signs up, or insert manually.';
    ELSE
        INSERT INTO public.platform_staff (user_id, role)
        VALUES (v_user_id, 'super_admin')
        ON CONFLICT (user_id) DO NOTHING;
    END IF;
END $$;
```

  <name>Task 2: Extend RLS on the 11 business tables (9 mutable get full CRUD, 2 immutable get SELECT only)</name>
  <files>supabase/migrations/20260506000001_admin_foundation.sql</files>
  <action>
Append to the SAME migration file from Task 1. For each business table, add admin RLS policies named `{table}_admin_*` that grant access when `public.is_platform_staff((SELECT auth.uid()))` is true. Use `DROP POLICY IF EXISTS` before each `CREATE POLICY` so re-running is safe.

**Pattern for mutable tables — apply to each:** organizations, profiles, organization_members, automation_templates, automations, automation_requests, subscriptions, notifications, invitations:

```sql
DROP POLICY IF EXISTS "{table}_admin_select" ON public.{table};
CREATE POLICY "{table}_admin_select"
    ON public.{table}
    FOR SELECT
    TO authenticated
    USING (public.is_platform_staff((SELECT auth.uid())));

DROP POLICY IF EXISTS "{table}_admin_insert" ON public.{table};
CREATE POLICY "{table}_admin_insert"
    ON public.{table}
    FOR INSERT
    TO authenticated
    WITH CHECK (public.is_platform_staff((SELECT auth.uid())));

DROP POLICY IF EXISTS "{table}_admin_update" ON public.{table};
CREATE POLICY "{table}_admin_update"
    ON public.{table}
    FOR UPDATE
    TO authenticated
    USING (public.is_platform_staff((SELECT auth.uid())))
    WITH CHECK (public.is_platform_staff((SELECT auth.uid())));

DROP POLICY IF EXISTS "{table}_admin_delete" ON public.{table};
CREATE POLICY "{table}_admin_delete"
    ON public.{table}
    FOR DELETE
    TO authenticated
    USING (public.is_platform_staff((SELECT auth.uid())));
```

**Important:** This is ADDITIVE. PostgreSQL RLS combines policies for the same command with OR — any matching policy permits access. Do NOT touch the existing org-scoped policies; clients keep their access exactly as before.

**Pattern for immutable tables (SELECT only, no admin INSERT/UPDATE/DELETE)** — apply to: automation_executions, chat_messages:

```sql
DROP POLICY IF EXISTS "{table}_admin_select" ON public.{table};
CREATE POLICY "{table}_admin_select"
    ON public.{table}
    FOR SELECT
    TO authenticated
    USING (public.is_platform_staff((SELECT auth.uid())));
```

For chat_messages: deliberately do NOT add an admin INSERT policy. The existing `chat_messages_insert_clients` constrains `sender_type = 'client'`. If admins ever need to send "AIDEAS-side" messages, that's a future feature using service_role (consistent with current architecture).
