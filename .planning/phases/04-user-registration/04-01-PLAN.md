  <action>
Create a new migration file `supabase/migrations/20260401000001_user_registration.sql` that does the following in order:

1. **Add columns to profiles:** `first_name TEXT`, `last_name TEXT`, `org_id UUID REFERENCES public.organizations(id)`. Keep existing `full_name` for backward compatibility.

2. **Update organization_members role CHECK constraint:** Drop existing `organization_members_role_check` constraint and recreate with `CHECK (role IN ('owner', 'admin', 'operator', 'viewer'))`.

3. **Create slug generation function:** `public.generate_org_slug(name TEXT) RETURNS TEXT` — converts to lowercase, replaces non-alphanumeric with hyphens, trims leading/trailing hyphens, loops with incremental counter if slug exists in organizations table. Use `LANGUAGE plpgsql`.

4. **Replace handle_new_user trigger function:** `CREATE OR REPLACE FUNCTION public.handle_new_user()` with `SECURITY DEFINER SET search_path = ''`:
   - Extract `company_name` from `NEW.raw_user_meta_data` with COALESCE fallback to `'My Organization'`
   - Generate slug via `public.generate_org_slug(company_name)`
   - INSERT into `public.organizations (name, slug)` RETURNING id
   - INSERT into `public.profiles (id, email, full_name, first_name, last_name, org_id)` — build `full_name` from CONCAT of first_name + last_name, extract first_name/last_name from metadata
   - INSERT into `public.organization_members (organization_id, user_id, role)` with role = 'owner'
   - Add EXCEPTION block that logs to raise a WARNING (so signup doesn't silently fail)

5. **Update seed data:** Update `supabase/seed.sql` to set `first_name`, `last_name`, and `org_id` on existing seeded profiles so `supabase db reset` still works cleanly. Also update the seeded `organization_members` entries to use 'owner' role for the first user in each org.

Note: The trigger fires on EVERY `auth.users` INSERT (including Google OAuth). For OAuth users, `company_name` will be NULL (user hasn't completed /complete-registration yet), so the COALESCE to 'My Organization' is critical. The /complete-registration Server Action (Plan 03) will UPDATE the org name afterward.

```typescript
import { z } from 'zod'

export const signupSchema = z.object({
  firstName: z.string().min(1, 'First name is required').max(50),
  lastName: z.string().min(1, 'Last name is required').max(50),
  companyName: z.string().min(1, 'Company name is required').max(100),
  email: z.string().email('Invalid email address').transform(val => val.toLowerCase().trim()),
  password: z.string()
    .min(8, 'Password must be at least 8 characters')
    .regex(/[A-Z]/, 'Must contain at least one uppercase letter')
    .regex(/[0-9]/, 'Must contain at least one number'),
  termsAccepted: z.literal(true, {
    errorMap: () => ({ message: 'You must accept the Terms of Service' }),
  }),
  locale: z.enum(['en', 'es']).default('en'),
  termsAcceptedAt: z.string().datetime().optional(),
  captchaToken: z.string().min(1, 'Verification required'),
})

export type SignupFormData = z.infer<typeof signupSchema>
```
