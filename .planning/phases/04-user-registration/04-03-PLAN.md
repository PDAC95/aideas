<task type="auto">
  <name>Task 1: Create Server Actions for signup and complete-registration</name>
  <files>web/src/lib/actions/auth.ts</files>
  <action>
Create `web/src/lib/actions/auth.ts` with 'use server' directive. Implement these Server Actions:

**1. `signUpWithEmail(formData: unknown)`:**
- Parse with `signupSchema.safeParse(formData)` — return `{ error: 'validation', issues }` on failure
- **Email normalization:** lowercase + trim (Zod transform handles this, but double-check)
- **Disposable email check:** Import domain list from `disposable-email-domains-js`. Extract domain from email. If disposable, return `{ error: 'disposable_email' }`
- **reCAPTCHA v3 server-side verification:** POST to `https://www.google.com/recaptcha/api/siteverify` with `secret` (from `process.env.RECAPTCHA_SECRET_KEY`) and `response` (captchaToken). If score < 0.5 or success === false, return `{ error: 'captcha_failed' }`
- **Duplicate email detection:** Call `supabase.auth.signUp(...)`. After response, check `data.user?.identities?.length === 0` — if so, return `{ error: 'email_exists', field: 'email' }`. This is the reliable Supabase pattern when email confirmation is enabled (Supabase returns a dummy user with empty identities instead of an error).
- **Signup call:** `supabase.auth.signUp({ email, password, options: { emailRedirectTo: process.env.NEXT_PUBLIC_SITE_URL + '/auth/callback?next=/verify-email', data: { first_name: firstName, last_name: lastName, company_name: companyName, locale, terms_accepted_at: termsAcceptedAt } } })`
- On Supabase error, return `{ error: error.message }`
- On success, return `{ success: true }`

**2. `completeRegistration(formData: { companyName: string })`:**
- Parse with `completeRegistrationSchema.safeParse(formData)`
- Get current user via `supabase.auth.getUser()`
- If no user, return `{ error: 'not_authenticated' }`
- Create admin client using `createClient` from '@supabase/supabase-js' with `SUPABASE_SERVICE_ROLE_KEY`
- Generate slug: call `supabase.rpc('generate_org_slug', { name: formData.companyName })` or do it in JS (simpler: let the DB trigger have already created the org with "My Organization", now UPDATE it)
- UPDATE `organizations` SET name = companyName, slug = new_slug WHERE id = user's org (lookup via organization_members)
- UPDATE `profiles` SET first_name, last_name from Google metadata if not already set
- Return `{ success: true }`

**3. `resendVerificationEmail(email: string)`:**
- Call `supabase.auth.resend({ type: 'signup', email })`
- Return `{ success: true }` or `{ error: error.message }`

Use the Supabase server client (`createClient` from '@/lib/supabase/server') for user-scoped operations. Use the admin client (direct `createClient` from '@supabase/supabase-js' with service role key) only for the org update in `completeRegistration`.

**Step 1: Wire signup form to Server Action:**

  7. If `success`, call `router.push('/verify-email?email=' + encodeURIComponent(email))`
