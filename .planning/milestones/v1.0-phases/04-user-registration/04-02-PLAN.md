**password-strength-bar.tsx** — "use client" component:
- Props: `{ password: string }`
- Compute strength score from password value:
  - Weak (red, `bg-red-500`): length < 8 OR missing uppercase OR missing number
  - Medium (yellow, `bg-yellow-500`): meets minimum (8+ chars, has uppercase, has number)
  - Strong (green, `bg-green-500`): meets minimum PLUS has special character AND length >= 12
- Render: horizontal bar (3 segments), filled segments colored by score, text label below
- Use `useTranslations('signup')` for Weak/Medium/Strong labels

**google-oauth-button.tsx** — "use client" component:
- Props: `{ redirectTo?: string }` (defaults to '/complete-registration' for new signups)
- On click: calls `supabase.auth.signInWithOAuth({ provider: 'google', options: { redirectTo: window.location.origin + '/auth/callback?next=' + redirectTo } })`
- Shows Google "G" icon (inline SVG or lucide doesn't have Google icon — use a simple SVG)
- Translated text: "Continue with Google"
- On error: show toast notification (use a simple div with setTimeout auto-dismiss after 5s, or sonner if available). Per CONTEXT.md: "Google OAuth errors: toast notification at the top that disappears in 5 seconds"

**language-switcher.tsx** — "use client" component:
- Small button showing "ES | EN" with the active locale highlighted (bold or different color)
- On click: sets `NEXT_LOCALE` cookie to the other locale, calls `router.refresh()`
- Reads current locale from `useLocale()` hook from 'next-intl'
- Styled as a subtle text button, not a prominent UI element

Use existing shadcn UI components (Button, Input, Label) wherever possible. Import icons from lucide-react.
