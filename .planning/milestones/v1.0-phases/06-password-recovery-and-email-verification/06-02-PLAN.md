1. **Add email verification gate** to `web/src/lib/supabase/middleware.ts`:

   Insert AFTER the existing "Protected routes - redirect to login if not authenticated" block and BEFORE the "Redirect logged-in users away from auth pages" block:

   ```typescript
   // Email verification gate — unverified users redirected to /verify-email
   if (
     user &&
     !user.email_confirmed_at &&
     !request.nextUrl.pathname.startsWith('/verify-email') &&
     !request.nextUrl.pathname.startsWith('/auth/') &&
     (request.nextUrl.pathname.startsWith('/dashboard') ||
       request.nextUrl.pathname.startsWith('/app/'))
   ) {
     const url = request.nextUrl.clone()
     url.pathname = '/verify-email'
     url.searchParams.set('email', user.email ?? '')
     return NextResponse.redirect(url)
   }
   ```

   Key design notes:
   - `user.email_confirmed_at` is populated by Supabase when verification link is clicked
   - Google OAuth users always have `email_confirmed_at` set (Google verifies emails), so this only gates email/password signups
   - Must exclude `/verify-email` and `/auth/` paths to prevent redirect loops
   - The `signInWithEmail` Server Action (Phase 5) already catches `email_not_verified` and redirects client-side — this middleware gate is defense-in-depth for direct URL access
