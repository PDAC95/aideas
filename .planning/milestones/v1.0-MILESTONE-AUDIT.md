## Tech Debt

### Phase 4: User Registration
- `/complete-registration` page has no server-side auth guard — renders then fails gracefully for unauthenticated users

### Phase 5: User Login
- `middleware.ts` → `proxy.ts` rename to fix build warning (tracked in deferred-items.md)

### Phase 6: Password Recovery
- 06-01-SUMMARY describes `requestPasswordReset` Server Action but actual implementation uses browser-side `supabase.auth.resetPasswordForEmail` — summary is inaccurate, no functional impact
- Error classification in `resetPassword` uses substring match on Supabase error messages — functional but brittle

### Cross-Phase
- VERIFY-04 email verification gate is middleware-only — dashboard layout checks auth but not `email_confirmed_at`. Low risk (middleware handles it correctly) but no defense-in-depth fallback.

**Total: 5 items across 4 areas — all non-blocking**
