### Open Issues (preexisting, NOT addressed in Phases 11 or 12)

#### HIGH-1: 'operations' category orphaned from catalog UI
- **Affected requirements:** CATL-01, CATL-02, I18N-01
- **Evidence:**
  - 8 seed templates with `category = 'operations'` in `supabase/seed.sql:721,735,749,763,777,791`
  - i18n missing: `web/messages/en.json` and `es.json` `dashboard.catalog.categories` block has 9 keys, **none for `operations`**
  - `web/src/components/dashboard/catalog-client.tsx:32-42` `CATEGORY_ORDER` does not contain `"operations"`
  - `web/src/app/(dashboard)/dashboard/catalog/page.tsx:62-72` translation map does not include `operations`
- **Impact:** Templates appear under "All" tab only; no dedicated filter; category badge has no label.

#### HIGH-2: 'agencias' industry orphaned from catalog UI
- **Affected requirements:** CATL-01, CATL-03, I18N-01
- **Evidence:**
  - ~48 seed templates include `'agencias'` in `industry_tags`
  - i18n missing: `dashboard.catalog.industries` block has 6 keys, **none for `agencias`**
  - `catalog-client.tsx:128-130` uses `Object.keys(translations.industries)` so chip is silently absent
- **Impact:** ~73% of catalog cannot be filtered by agency-specific use cases.

#### MEDIUM-1: updateAutomationStatus missing org ownership check
- **Affected requirements:** AUTO-06
- **Evidence:** `web/src/app/(dashboard)/dashboard/automations/[id]/actions.ts:6-23` updates by `automationId` only; no org membership check, no role check. Inconsistent with Phase 12 settings actions which explicitly verify `organization_members.role`.
- **Impact:** Defense-in-depth gap — relies solely on RLS at DB level. A user authenticated to org A could pause/resume automations in org B if RLS policy is permissive or a bug is introduced.

### Open Issues (i18n leakage)

#### LOW-1: Hardcoded "Just now" in automations detail
- **Affected requirements:** AUTO-04, I18N-01
- **File:** `web/src/app/(dashboard)/dashboard/automations/[id]/page.tsx:24`
- **Detail:** `if (seconds < 60) return "Just now";` — Spanish users see EN. Compare home page which uses `t("timeAgo.now")`.

#### NEW-LOW-1: NotificationBell formatRelativeTime bypasses i18n
- **Affected requirements:** NOTF-02, I18N-01
- **File:** `web/src/components/dashboard/notification-bell.tsx:20-26`
- **Detail:** Hardcoded `"now"`, `"m"`, `"h"`, `"d"` strings. Surfaces now that Phase 12 added a fully wired locale switch — Spanish users see EN time abbreviations in the notification dropdown.

### Open Issues (performance)

#### LOW-2: Redundant notifications query in fetchDashboardData
- **File:** `web/src/lib/dashboard/queries.ts:49-54`
- **Detail:** Notifications fetched but `dashboard/page.tsx:60` no longer destructures them. Layout fetches its own copy. Result fetched and silently discarded.
- **Impact:** Performance only — not a bug.
